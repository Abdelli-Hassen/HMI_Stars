\chapter*{General introduction}
\addcontentsline{toc}{chapter}{General Introduction}
In today's highly dynamic and competitive business environment responsiveness and the smooth flow of information are critical success factors for enterprises. Consulting and administrative support companies work closely with business managers and help them handle important operations However, many of these companies still use scattered communication methods like emails, phone calls, and unsecured messaging apps. This makes it hard to track documents properly and slows down decision-making.

This project addresses this issue within the consulting enterprise \textbf{HMI Stars Consulting}.
Its goal is to design and implement \textbf{HMI Stars}, a unified application ecosystem that modernizes the client–enterprise relationship.
The main objective is to centralize administrative document transmission, employee management, activity tracking, and message exchanges within a single real-time software system.

To meet this need, we structured our development around a monorepo architecture utilizing the \textbf{Flutter} framework for both web and mobile frontend applications, and \textbf{Supabase} for data persistence, secure authentication, and real-time synchronization.

This report is organized into three main chapters according to the Unified Process methodology guidelines:
\begin{itemize}
\item \textbf{Chapter 1} introduces the host organization, defines the problem, analyzes the existing system, and reviews related work on business communication systems.

\item \textbf{Chapter 2} covers the system design and development, including the Inception, Elaboration, and Construction phases, with UML modeling, database design, and system architecture.

\item \textbf{Chapter 3} presents the implementation phase, focusing on system deployment, functional testing, interface demonstrations, and usability evaluation.

\end{itemize}
