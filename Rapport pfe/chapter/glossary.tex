\chapter*{Glossary}
\addcontentsline{toc}{chapter}{Glossary}
\begin{spacing}{1.4}
\noindent\textbf{Firebase Cloud Messaging (FCM):} A cross-platform cloud solution that allows developers to reliably send notifications and messages to mobile and web client applications.

\noindent\textbf{Flutter:} A Google open-source software development kit (SDK) used to build cross-platform applications for mobile, web, and desktop from a single codebase.

\noindent\textbf{JSON Web Token (JWT):} An open standard (RFC 7519) that defines a compact and self-contained way for securely transmitting information between parties as a JSON object.

\noindent\textbf{Monorepo:} A software development strategy where code for multiple projects, libraries, or applications is stored within the same version control repository.

\noindent\textbf{One-Time Password (OTP):} A secure, automatically generated numeric code that authenticates a user for a single login session or transaction, preventing replay attacks.

\noindent\textbf{PostgreSQL:} A powerful, open-source object-relational database system (ORDBMS) known for its reliability, feature robustness, and performance.

\noindent\textbf{REST (Representational State Transfer):} An architectural style for designing networked applications that uses HTTP requests to GET, PUT, POST, and DELETE data.

\noindent\textbf{Row Level Security (RLS):} A database security mechanism in PostgreSQL that restricts the rows returned by a query or affected by data modification commands based on user permissions or context.

\noindent\textbf{SME (Small and Medium-sized Enterprises):} Independent enterprises which employ fewer than a given number of employees, typically small businesses.

\noindent\textbf{Supabase:} An open-source alternative to Firebase that provides back-end services, including a real-time PostgreSQL database, authentication, storage, and auto-generated RESTful APIs.

\noindent\textbf{Two Track Unified Process (2TUP):} A software development methodology characterized by a dual track approach, dividing work into a functional track (use cases) and a technical track (architecture and constraints).

\noindent\textbf{WebSockets:} A communication protocol providing full-duplex communication channels over a single TCP connection, enabling real-time data flow between a client and a server.
\end{spacing}
