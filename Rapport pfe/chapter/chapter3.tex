\section{Introduction}
This final chapter focuses on the realization and testing of the HMI Stars ecosystem. We cover the Transition phase, compare the specifications with the actual system, detail the execution environment, and present functional test results and usability evaluations.

\section{Phase of Transition}
During the transition phase, the system was prepared and packaged for deployment and local evaluation . \\The web administration platform was configured and deployed on a local web server (utilizing local port binding and hot-reload tooling for simulation) , allowing internal managers to access it directly within the intranet environment .\\ In parallel , the mobile client application was compiled into release-ready binaries (specifically , an optimized Android package/APK) to facilitate local device and\\ emulator testing .\\ Both components interface seamlessly with the remote Supabase cloud database\\ instance .

\subsubsection{Advanced Security and Authentication Workflows}
To provide a secure and user-friendly experience, we updated the authentication system by replacing traditional email magic links with a native, in-app 6-digit One-Time Password (OTP) verification flow. When a client manager registers or tries to log in with an unconfirmed email, the system catches the exception and prompts them with a verification screen to input their OTP. Similarly, during password recovery, the user enters their email, receives a code, verifies it in-app, and resets their password without encountering browser redirection loops. 

Additionally, we added strict security checks on the profile settings page. Before updating sensitive information like passwords or emails, the app silently reauthenticates the user with their current password. Email changes use a dual-OTP sequence to verify both the old and new email addresses, protecting the account from unauthorized takeover.

\subsubsection{Bilingual Email Templates and Multi-Channel Notification Subsystem}
To guarantee a professional and cohesive user experience, we customized the automated email notification templates dispatched by the Supabase authentication service. Rather than sending plain, default system emails, we designed premium responsive HTML email templates. As specified in the database and security configurations, these templates feature a unified layout with French copy at the top and English copy below, ensuring full bilingual support. Structurally, the emails are styled using clean HTML/CSS containing a corporate header, the HMI Stars logo, and distinct card components highlighted with the signature gold-brown accent (\#b4975a). This includes:
\begin{itemize}
    \item \textbf{Confirm Sign Up}: Delivers the 6-digit registration confirmation OTP with the subject \textit{Confirmez votre inscription | Confirm Sign Up}.
    \item \textbf{Reset Password}: Dispatches recovery OTPs with the subject \textit{Réinitialisation de votre mot de passe | Reset Password}.
    \item \textbf{Email Change Confirmation}: Drives the dual-OTP verification steps with the subject \textit{Confirmez le changement d'email | Confirm Email Change}.
    \item \textbf{Security Notifications}: Dispatches instant notices confirming completed modifications to passwords or emails. Additionally, phone number changes trigger an immediate notification email containing both old and new numbers to maintain an audit trail.
\end{itemize}

Complementing the email alerts, we implemented a dual-mode notification architecture for in-app events:
\begin{itemize}
    \item \textbf{FCM Push Notifications}: Real-time push alerts are routed via Firebase Cloud Messaging (FCM). When administrative updates are triggered, the server resolves the device token stored in the database (\texttt{entreprises.jeton\_notification}) to deliver background notification packets
    instantly.
    \item \textbf{Gold-Themed Toast Notification Engine}: For foreground user feedback, the interface features a custom toast notification manager (\texttt{ToastUtils.show}). Positioned non-intrusively in the bottom-right corner of the desktop viewport, it displays state changes (such as database sync confirmations or validation errors) using a floating card styled with a solid gold border and a smooth bounce slide-in animation driven by a cubic bezier curve (\textit{Curves.easeOutBack}).
\end{itemize}


\subsubsection{Interactive Document Scanner Subsystem}
To make document uploads easier, we integrated an interactive mobile document scanner. This tool allows client managers to snap photos of physical paper records (such as receipts, tax declarations, or KBIS extracts) and crop them using a boundary-detection overlay. 

To prevent high-resolution photos from causing performance lags or Application Not Responding (ANR) states, the application downscales camera images natively to a maximum width of 1200 pixels and height of 1600 pixels at 85\% quality. This optimization speeds up the perspective cropping calculations, completing them in less than a second. We also configured the scanner to use fullscreen immersive mode during active cropping. This aligns the cropping coordinates perfectly with the device screen size, preventing the bottom area of cropped documents from being cut off.

\subsubsection{Disciplinary Warnings and Document Templates Subsystem}

\section{Conclusion}
The testing and evaluation phases confirmed that the HMI Stars ecosystem is stable, secure, and fully meets all initial project requirements. The mobile application successfully delivers employee management, attendance tracking, leave requests, real-time messaging, document sharing with scanning capabilities, and disciplinary warning templates. The web platform provides a comprehensive administration workspace with company management, employee registries, document archives, warning systems, notes and reminders, and attendance export capabilities. Both components support bilingual interfaces (French and English) and adaptive themes (light and dark), ensuring accessibility and comfort for all users. The transition phase successfully configured and launched all components within their designated execution environments, confirming readiness for production deployment.
