\chapter{System realization}

\section{Introduction}
In this final chapter, we detail the implementation, deployment, and validation of the HMI Stars platform. We describe the transition phase, compare the initial specifications against our actual implementation, define the execution environment, and evaluate both system functionality and overall usability.

We also highlight how the mobile client application and the web administration dashboard support dynamic appearance modes (switching between light and dark themes) and fully localized interfaces (supporting French and English language options) across all screens and user components.

\section{Phase of transition}
During the transition phase, we packaged and configured the entire system for deployment, local testing, and production hosting. We verified the web administration panel on a local development server to simulate internal workflows, and deployed the production version to the cloud using the \textbf{Vercel} hosting platform for remote availability. Simultaneously, we compiled the mobile client application into an optimized Android Package (APK) for testing on physical devices and emulators. Both client frontends communicate directly with the remote Supabase cloud database.

\section{Execution environment}
We evaluated and tested the system across multiple operational platforms:
\begin{itemize}
    \item \textbf{Web Administration Portal (Admin Client):} Executed and tested using Microsoft Edge and Google Chrome web browsers.
    \item \textbf{Mobile Client Application:} Deployed and run on both Android emulators and physical Android devices.
    \item \textbf{Backend Infrastructure:} Powered locally and by a cloud-hosted Supabase project.
\end{itemize}

\paragraph{Accounts Management}
Figure \ref{fig:web_accounts_management} presents the redesigned user account management panel. This interface provides administrators with comprehensive controls to manage and audit user access, including handling unverified registrations:
\begin{itemize}
    \item \textbf{User Search \& Filter:} A real-time search input enables filtering accounts by name or email address immediately.
    \item \textbf{Verification Status Badges:} Accounts whose email address has not yet been validated are marked with a distinct amber "Non vérifié" (Unverified) status badge, allowing admins to track registration progress.
    \item \textbf{Manual Email Verification:} A dedicated envelope action button enables administrators to manually confirm the user's email address and bypass the OTP verification process when necessary.
    \item \textbf{Access Toggle Switches:} Interactive switches allow administrators to temporarily enable or disable access for individual accounts.
    \item \textbf{Role Configuration Dropdowns:} Role selectors (e.g., Admin, Secrétaire) are integrated directly in the list, allowing quick role adjustments.
    \item \textbf{Impersonation (Login As):} A shortcut button allows administrators to log in as the target user (impersonation mode) for troubleshooting and assistance.
    \item \textbf{Deletion:} Dedicated action buttons enable permanent deletion of user credentials from the authentication system.
\end{itemize}

\section{Conclusion}
The validation and evaluation phases demonstrate that the HMI Stars ecosystem is stable, secure, and complete. The mobile application delivers functional modules for personnel tracking, attendance logging, absence recording, real-time communication, document scanning, and warning template customization. Similarly, the web platform provides administrators with directories for client companies, active registries, document archiving systems, customized disciplinary workflows, global notes, and bilingual attendance export utilities. Both client frontends support bilingual translation options and adaptive light/dark visual themes to optimize accessibility. Finally, our transition phase deployment confirms that all components operate reliably within their production-ready execution environments.
