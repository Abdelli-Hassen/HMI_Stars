If your goal is the **absolute best possible PFE report** (not just a passing one), think of it as being evaluated on 5 axes:

1. Scientific quality
2. Technical quality
3. Professional presentation
4. Academic integrity
5. Defense readiness

The following checklist combines common Tunisian university requirements (ISG, ISET, engineering schools, universities) and best practices used by students who obtain excellent grades. ([ESCT][1])

# 1. Structure of the Report

A top-quality PFE should generally contain:

### Preliminary Pages

* Cover page (official university template)
* Dedication (optional)
* Acknowledgements
* Abstract / Résumé (French or English)
* Arabic summary (if required)
* Keywords
* Table of contents
* List of figures
* List of tables
* List of abbreviations/acronyms
* Glossary (if needed)

### Main Body

#### General Introduction

* Context
* Problem statement
* Objectives
* Methodology
* Report organization

#### Chapter 1: Company Presentation

* Company overview
* Organizational structure
* Business activities
* Department where internship was done

#### Chapter 2: Existing System Study / Literature Review

* Current situation
* Problems identified
* Existing solutions
* Comparative study
* Critique of existing system

#### Chapter 3: Analysis and Design

For IT projects:

* Requirements specification
* Functional requirements
* Non-functional requirements
* Use cases
* UML diagrams
* Architecture

For management/accounting projects:

* Theoretical framework
* Methodology
* Data collection
* Analysis approach

#### Chapter 4: Implementation / Practical Work

* Technologies used
* Development process
* Screenshots
* Tests
* Results

#### General Conclusion

* Achievements
* Limitations
* Future improvements

### End Matter

* Bibliography
* Webography
* Appendices
* Plagiarism declaration (if required)

([Studocu][2])

---

# 2. Formatting Rules

These details often affect grades more than students think.

### Font

Usually:

* Times New Roman
* Size 12 body text
* Size 14–16 chapter titles

### Line Spacing

* 1.5 lines

### Margins

Typical:

* Left: 3 cm
* Right: 2.5 cm
* Top: 2.5 cm
* Bottom: 2.5 cm

### Alignment

* Fully justified text

### Page Numbers

* Consistent throughout
* Preliminary pages may use Roman numerals

### Chapter Titles

* Consistent numbering:

  * Chapter 1
  * 1.1
  * 1.1.1

Never mix styles.

([Studocu][2])

---

# 3. Writing Quality Rules

### Avoid

* Long paragraphs
* Informal language
* Personal opinions
* SMS language

### Prefer

* Short clear sentences
* Academic style
* Objective statements

Bad:

> I think this solution is very good.

Good:

> The proposed solution improves processing efficiency by 35%.

### Consistency

Choose one language style and keep it.

Example:

* Do not mix French and English terminology randomly.

---

# 4. Figures and Screenshots

Every figure must have:

* Figure number
* Caption
* Source (if not yours)

Example:

Figure 3.4: Authentication Interface

Rules:

* High resolution only
* No blurry screenshots
* Same size/style across report

([Studocu][2])

---

# 5. Tables

Every table must have:

* Number
* Title
* Source if needed

Example:

Table 2.3: Comparison of Existing Solutions

A strong comparison table impresses juries.

---

# 6. Bibliography Rules

Use a consistent citation style:

* APA
* IEEE
* Harvard

Do not mix styles.

Every citation inside the text must appear in bibliography.

Every bibliography entry should be cited in the report.

---

# 7. Plagiarism Rules

Major mistakes:

* Copy-pasting from old PFE reports
* Copy-pasting from websites
* Copy-pasting AI-generated text

Always:

* Cite sources
* Rewrite in your own words
* Add your own analysis

Many institutions verify originality before acceptance. ([Studocu][2])

---

# 8. IT Project-Specific Rules

If your PFE is software-related:

Include:

* Use Case Diagram
* Class Diagram
* Sequence Diagram
* Deployment Diagram
* Architecture Diagram
* Database Schema
* Test Cases

Show:

* Problem
* Design
* Development
* Validation

Do not only show screenshots.

---

# 9. What Makes a Report Look Professional

### Use automatic Word features

* Table of contents
* Figure index
* Table index
* Captions
* Cross-references

Never type page numbers manually.

### Consistent Design

Same:

* Fonts
* Headings
* Spacing
* Colors

### Headers & Footers

Include:

* University
* Report title
* Chapter name

if allowed.

---

# 10. Common Reasons Students Lose Points

* Spelling mistakes
* Poor formatting
* Inconsistent numbering
* Missing references
* Low-quality screenshots
* No critical analysis
* Too much theory
* Too little personal contribution
* Weak conclusion
* Copy-paste content

---

# 11. What Gets Excellent Grades (16–20/20)

A jury is usually impressed when:

✓ Problem is clearly defined

✓ Report looks professionally formatted

✓ Diagrams are clean

✓ References are properly cited

✓ Results are measured and analyzed

✓ Student explains decisions and trade-offs

✓ No obvious plagiarism

✓ Strong conclusion and future work

✓ Presentation matches the report

✓ Student fully masters every page of the document
